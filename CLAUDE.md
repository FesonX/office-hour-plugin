## Changelog Guidelines

The changelog is for humans — keep it short and outcome-focused. Implementation details belong in commit messages.

**Rules:**
1. **Group by type** — use `Added`, `Changed`, `Fixed`, `Removed` (not `Feature`, `Refactor`)
2. **One line per entry** — aim for ≤100 characters; no file paths, no code snippets, no error codes
3. **Write from the user's perspective** — what does the skill user experience differently?
4. **Lead with a verb, not a component** — "Block deployment until AI test passes" not "Step 8: replace passive hint with blocking POST"
5. **No internal cross-references** — don't cite file names like `claude-ai-integration.md`; describe the capability instead
